# Тестовое задание: Пайплайн аналитики маркетплейсов с ИИ-классификацией и интеграцией с CRM 

## 📖 Бизнес-контекст
Компании необходимо автоматизировать процесс мониторинга конкурентов и поиска нишевых товаров. Мы собираем каталоги маркетплейсов, с помощью ИИ определяем ценовой сегмент товаров (чтобы не анализировать «премиум» и «эконом» в одной куче) и автоматически ставим задачи менеджерам в CRM для проработки самых интересных позиций. 

Простыми словами: Нужно написать программу, которая сама зайдёт на маркетплейс, соберёт товары, с помощью нейросети определит какие из них дешёвые, а какие дорогие, и создаст задачи в CRM для менеджеров.

## 📋 Функциональные требования 

### 1. Парсинг маркетплейса (сбор товаров) 

Что нужно сделать:
- Выбрать одну категорию товаров (например, "Смартфоны" или "Кроссовки")
- Собрать 10 000 товаров из этой категории 

Технические детали (с пояснениями): 

✅ Пагинация — это когда товары выдаются не все сразу, а страницами. Например, на первой странице 100 товаров, на второй ещё 100, и так далее. Нужно уметь переключаться между страницами. 

✅ Обработка rate-limit (429 Too Many Requests) — маркетплейсы ограничивают количество запросов. Если вы делаете слишком много запросов в минуту, сервер вернёт ошибку 429. Нужно:
- Отлавливать эту ошибку
- Ждать указанное время (или использовать экспоненциальную задержку: 1 сек → 2 сек → 4 сек → 8 сек)
- Повторять запрос 

✅ Graceful degradation (плавная деградация) — если маркетплейс начал отдавать ошибки или упал, программа не должна падать с ошибкой. Вместо этого она должна:
- Сохранить всё, что успела собрать
- Записать в лог, что произошло
- Корректно завершить работу 

✅ DEMO_MODE — добавьте переменную окружения DEMO_MODE=true, чтобы при локальном тестировании собирать не 10 000 товаров, а только 50-100. Это ускорит тесты и сэкономит ресурсы. 

--- 

### 2. Нормализация и обогащение данных (LLM) 

Что нужно сделать:
- Привести все собранные данные к единому формату (схеме)
- С помощью нейросети определить ценовой сегмент каждого товара 

Единая схема данных (Pydantic):
class Product(BaseModel):
    id: str              # ID товара на маркетплейсе
    name: str            # Название товара
    price: float         # Цена
    currency: str        # Валюта (RUB, USD, etc.)
    url: str             # Ссылка на товар
    category: str        # Категория
    description: str     # Описание (если есть)
    collected_at: datetime  # Когда собрали
    segment: str         # Сегмент: Эконом / Стандарт / Премиум (добавит LLM)

Этап ИИ-классификации:
- Прогнать названия и описания товаров через LLM
- LLM должна присвоить каждому товару один из трёх сегментов:
  - Эконом — дешёвые товары
  - Стандарт — товары среднего ценового диапазона
  - Премиум — дорогие товары 

⚠️ Важное требование к архитектуре:
Делать 10 000 отдельных запросов к LLM — это очень дорого и долго. Нужно реализовать один из вариантов:
